Dynamic Profile Modal

One of the many tasks that was given to me was the development of a profile modal for a logged in user. The profile would display their name, photo, role in the *ship Startup event, registered email, profile creation date/time, last sign in date/time and their system role. All this information could not possibly be hardcoded for every existing user, even more so for new users. Therefore, I pulled the data from the firestore database and used embedded javascript to insert the required data into HTML code using innerHTML. This property would then be inserted into the main HTML page at runtime.

Test Plans

A crucial aspect of an agile software development process involves continuous testing of software components after they have been built. Our task as a group was to create a test plan as well as carry out these tests on the entire web app. This included login verification, correct data displayed on all pages, responsive design, among others. Tests were created by writing, for each test, a case type, test case, test data, expected outcome, actual outcome, result (pass/fail), any issues (if any), as well as actions to take and asignee for this task. This allowed us to see exactly what we needed to fix before the launch or identify anything important that was overlooked.

SecuriT

This is a software engineering group project that is currently ongoing. We have chosen to develop a parental control app for android mobiles, using Java as the main programming language. Using the agile method, we are learning about the process of developing an app by completing each of the various stages such as market research, design, testing and coding. So far we have completed:
- a market study that has lead to finding stakeholers for our app
- concept design models(UML User Case, Sequence and Activity diagrams) for our app and each stakeholder
- low and high fidelity prototypes (using Adobe XD)
- collected feedback from our stakeholders for both prototypes and analysed this data so we are more onformed on how to improve our design and proceed with development
This process also teaches us how to correctly write technical specifications and reports in a way that is detailed and informative. Our app with consist of features such as location, scheduling, panic button (for emergencies) and app access to restrict certain apps for a period of time.
